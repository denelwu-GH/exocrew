# Contributing

Contributions should make ExoCrew more useful without turning it into an unbounded collection of advice.

## Good contributions

- A new gate backed by a real failure or near miss
- A clearer stop condition for a risky workflow
- A smaller, more reliable template or script
- A benchmark task with an independent acceptance rubric
- A fix for an installation, validation, or documentation defect

## Contribution requirements

1. Explain the problem and the evidence that it is real.
2. State the affected skill and why existing guidance is insufficient.
3. Keep durable truth in one place; link instead of duplicating it.
4. Add or update deterministic validation when possible.
5. Run `npm test`.
6. Do not include private data, private paths, credentials, production identifiers, or source-project business rules.
7. Label examples as examples and scaffolds as scaffolds.

## Pull request summary

Include:

- current behavior
- gap
- change
- verification
- residual risk
- rollback or removal path
